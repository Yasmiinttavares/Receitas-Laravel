# Roteiro de Apresentação

## 1. Contexto do projeto
- Apresentar o objetivo da aplicação: gerenciar receitas.
- Destacar que o sistema possui autenticação simples, CRUD de receitas e ambiente de homologação/prod.

## 2. Funcionalidades implementadas
- Cadastro e login de usuário.
- Cadastro, listagem, edição e exclusão de receitas.
- CI/CD com testes automatizados e deploy via Docker.

## 3. Arquitetura e infraestrutura
- Laravel como framework backend.
- PostgreSQL como banco de dados.
- Docker Compose para ambientes homolog/prod.
- GitHub Actions para CI/CD.

## 4. Demonstração
- Mostrar login e cadastro.
- Mostrar criação e listagem de receitas.
- Mostrar acesso via Docker e URLs de homolog/prod.

## 5. Pontos de melhoria
- Expandir testes de cobertura.
- Adicionar autenticação mais robusta no futuro.
- Melhorar observabilidade e monitoramento.
